# 로그인 로직 구현

목표: 구글/카카오/네이버 로그인 로직 구현

## 프로젝트 로직 파악

apis/moviemang - axios logic setup(http://35.203.6.209:8000/auth/) 에 요청

data/user - 로그인/회원가입에 따른 유저 상태 처리

```
유저가 로그인 버튼 클릭 시 -> action dispatch -> axios logic triggered -> response saved in redux state(user)
-> check user state
```

이런 로직으로 구현하면 될 듯 하다.

## 구현

1. user actions 에 snsLogin 함수 생성
2. dispatch함수에 snsLogin({code, loginType}) 형태로 전달해서 값 return
3. 돌아온 값에 따라서 분기처리 - message 에 따른 분기처리
   3-1. message = '닉네임을 설정해주세요.' 인 경우, /nickname으로 라우팅, nickname에서 새롭게 로직 구현

   - nickname으로 라우팅 처리하는 로직을 어디에서 처리하는게 적절할지? 각 페이지? 로그인 페이지 useEffect?
     -> 우선 로그인 페이지 useEffect로 구현하였음
     -> 유저 정보 DB에 없는 경우, 닉네임을 포함한 값을 보내야하는데 loginType을 백엔드에서 다시 받아서 처리하는게 어떨까 싶어 문의한 상태

     3-2. message = '로그인 성공'인 경우, /으로 라우팅(로그인 성공에 대한 UI 처리 필요?)

4. 닉네임 생성 페이지로 라우팅 되었을 때, 값을 받아서 /auth/user로 다시 요청해서 로그인 하는 로직 필요함

## 문제점

구현하면서 비동기 인증 / 리덕스 / 라우팅을 결합해서 구현하는 것이 쉽지 않다는 것을 알게 되었다.

정확하게 상태가 어떤 시점에서 업데이트 되는지 알지 못해서 더욱 그러하다

미들웨어가 왜 필요한지 직접 느끼게 되었음
-> 라우팅 하려는데, 안돼...그 안에서 상태를 볼 수가 없어...

자세히 기록해보자...

카카오 로그인(snsLogin) dispatch => 처리 도중에 과정에서 처리된 state 에 접근하려고 해도 안됨 undefined

이런 문제를 redux thunk 등 미들웨어를 통해서 해결해야 되는 듯 하다.
